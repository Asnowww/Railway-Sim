最简启动 1 列车：

```bash
# 1. 启动中央系统，启用外部车辆运行时
cd /Users/fangablt/Applications/EngineeringWorks/小学期/project/backend
mvn spring-boot:run -Dspring-boot.run.arguments="--railway.simulation.vehicle-runtime.mode=EXTERNAL_HTTP"
```

另开终端：

```bash
# 2. 启动车辆运行时服务 9300
cd /Users/fangablt/Applications/EngineeringWorks/小学期/project/vehicle-runtime-service
mvn spring-boot:run
```

再另开终端：

```bash
# 3. 由车辆仿真系统发起列车启动，并注册到中央
curl -X POST http://127.0.0.1:9300/vehicle-runtime/trains/launch \
  -H 'Content-Type: application/json' \
  -d '{
    "trainId": "TR-003",
    "linkId": 12,
    "offsetMeters": 640.0,
    "direction": "DOWN",
    "registerWithCentral": true,
    "reason": "demo launch",
    "traceId": "runtime-train-003"
  }'
```

验证：

```bash
curl http://127.0.0.1:8080/api/trains/TR-003
curl http://127.0.0.1:8080/api/vehicle/runtime-health
curl http://127.0.0.1:9300/vehicle-runtime/instances
```

发送 PLC 控制命令前必须完成上述 launch；9300 会拒绝不存在的列车实例。浏览器/人工联调使用 8080 的结构化入口，8080 会自动编码 46 字节 PLC 载荷并封装 RSIM 聚合帧：

```bash
curl -X POST http://127.0.0.1:8080/api/vehicle/driver-cabs/TR-003/plc-input \
  -H 'Content-Type: application/json' \
  -d '{"keySwitchLocked":false,"directionHandleState":"ZERO","masterHandleState":"ZERO","tractionNotchPercent":0,"brakeNotchPercent":0}'

# 服务间二进制入口只接受已经封装好的RSIM帧，不接受裸46字节PLC文件
curl -X POST http://127.0.0.1:9300/vehicle-runtime/peripherals/frame \
  -H 'Content-Type: application/octet-stream' \
  --data-binary @rsim-plc-input.bin
curl http://127.0.0.1:9300/vehicle-runtime/peripherals/status

# 信号侧状态投影与PLC显示输出仍从8080读取
curl http://127.0.0.1:8080/api/vehicle/driver-cabs/TR-003/state
curl http://127.0.0.1:8080/api/vehicle/driver-cabs/TR-003/plc-output
```

如果还想让它开始 tick：

```bash
curl -X POST http://127.0.0.1:8080/api/simulation/start
```

这条链路就是：`9300 launch -> 启动车辆仿真实例 -> 唤醒车辆控制实例 -> 注册中央 -> PLC输入经8080封装RSIM聚合帧 -> 9300仲裁 -> 中央后续 tick 调 9300`。网络屏、信号屏和视景系统只接收中央输出，不进入该输入链路。

每个tick使用中央 `DispatchService.simulationRunId`作为唯一运行主键。9300转发9200时必须携带同一runId；reset后旧run记为`CANCELLED_BY_RESET`，新run从`CREATED`开始，旧快照不删除。
