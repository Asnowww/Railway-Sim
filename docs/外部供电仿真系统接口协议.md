# 外部供电仿真系统接口协议

## 1. 目标

中央系统通过HTTP管理外部供电仿真的拓扑和设备操作，但不重新计算车辆供电能力。9200是网压、受流侧可用功率和再生吸收能力的权威；拆分部署中9300是唯一车辆负荷写入方，8080只读取状态镜像。外部供电系统负责：

- 牵引变电所设备
- 接触轨隔离开关
- 杂散电流监测点

## 2. 接口

### 2.1 `POST /power-network/bootstrap`

中央系统启动后发送虚拟电网拓扑、变电所设备、供电分区绑定、隔离开关和杂散监测点。

### 2.2 `GET /power-network/state`

外部系统返回：

- `substations[]`
- `thirdRailSections[]`
- `isolators[]`
- `strayCurrentMonitors[]`
- `heartbeatStatus`
- `dataQuality`
- `sourceTimestamp`

### 2.3 `POST /power-network/state/query`

兼容模式下写入区段负荷并返回更新后的供电状态：

- 非拆分部署的`LOCAL`、`DUAL_SHADOW`模式可由中央`PowerIntegrationService`调用。
- 拆分部署不调用该写接口，避免和9300重复提交负荷。

请求包含 `sectionLoads[]`，每项至少包括 `powerSectionId`、`trainIds`、`tractionPowerWatts`、`regenPowerWatts` 和 `currentAmps`。

### 2.4 `POST /power-network/constraints/query`

9300在车辆物理步前提交全车位置，9200返回每车所在供电段、网压、受流侧可用功率、再生接收预算和受流状态。该接口只读当前供电状态，不写车辆负荷。

### 2.5 `POST /power-network/step`

拆分部署的权威闭环写接口。请求必须包含：

- `tick`
- `simulationTimeSeconds=tick×0.1`
- `stepSizeSeconds=0.1`
- 完整`sectionLoads[]`
- 物理步后的`trainPositions[]`

9200把`T(n)`负荷求解为下一周期约束。同tick重复请求返回同一缓存结果，旧tick返回HTTP 409，不允许对同一车辆周期重复写负荷。响应中的`regenPowerAvailableWatts`按同段牵引电功率预算和车辆数等分；区段状态同时给出`absorbedRegenWatts`、`unabsorbedRegenWatts`和`regenBudgetWatts`。

### 2.6 `POST /power-network/operations`

中央系统下发操作命令，支持：

- 隔离开关分合
- 直流快速断路器分合/跳闸恢复
- 整流器停运/恢复
- 排流柜状态切换

### 2.7 `GET /power-network/events`

返回最近事件列表，供中央系统监控与审计消费。

## 3. 当前实现

当前仓库`power-network-service/`是独立FastAPI供电仿真服务。它支持：

- bootstrap 后维护内存态设备模型
- 返回中央系统可消费的全量设备快照
- 根据操作命令更新设备状态
- 用显式规则输出杂散电流风险等级
- 按供电段聚合多车电网侧负荷并计算电压下降
- 计算同段回馈吸收预算、实际吸收和未吸收功率
- 固定100 ms显式耦合、重复tick幂等和乱序保护
