# 外部供电仿真系统接口协议

## 1. 目标

中央系统通过 HTTP 与外部供电仿真系统交互。中央系统负责车辆约束与供电能力计算，外部系统负责设备级权威状态，包括：

- 牵引变电所设备
- 接触轨隔离开关
- 杂散电流监测点

## 2. 接口

### 2.1 `POST /power-network/bootstrap`

中央系统启动后发送虚拟电网拓扑、变电所设备、供电分区绑定、隔离开关和杂散监测点。

### 2.2 `GET /power-network/state`

外部系统返回：

- `substations[]`
- `thirdRailSections[]`
- `isolators[]`
- `strayCurrentMonitors[]`
- `heartbeatStatus`
- `dataQuality`
- `sourceTimestamp`

### 2.3 `POST /power-network/operations`

中央系统下发操作命令，支持：

- 隔离开关分合
- 直流快速断路器分合/跳闸恢复
- 整流器停运/恢复
- 排流柜状态切换

### 2.4 `GET /power-network/events`

返回最近事件列表，供中央系统监控与审计消费。

## 3. 当前脚手架

当前仓库已提供 `power-network-service/` 目录下的最小 Python 服务，可用于联调和接口验证。它支持：

- bootstrap 后维护内存态设备模型
- 返回中央系统可消费的全量设备快照
- 根据操作命令更新设备状态
- 用显式规则输出杂散电流风险等级
