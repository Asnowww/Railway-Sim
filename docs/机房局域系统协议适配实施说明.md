# 机房局域系统协议适配实施说明

## 1. 落地边界

现场协议按职责拆分到两个后端：中央/信号 `backend:8080` 保留信号 UDP、网络屏/信号屏和 PLC 显示输出；车辆管理 `vehicle-runtime-service:9300` 是 PLC 46 字节控制输入的唯一执行入口。中央不另起总控数据库节点服务。

实时报文收发不依赖数据库写入成功。`protocol_packet_log` 只做旁路诊断，写库失败时记录降级告警，现场链路继续运行。这不代表危险业务操作可绕过审计：故障注入/清除和供电操作必须先同步写 `operation_log`，失败返回 503。

协议依据为《轨交多系统平台接口协议汇总20260630》和《704对接可能出错的点》。两者冲突时，以补充文档的现场抓包和端点修正为准。

## 2. 启用方式

默认配置全部关闭：

```yaml
railway:
  simulation:
    localnet:
      enabled: false
      signal:
        enabled: false
      driver-cab:
        enabled: false
      power-points:
        enabled: false
```

机房联调时，把 `config/localnet-site-example.yml` 复制到本地 `application-local.yml`。该站点示例已启用司机台总开关和司机台链路，但不会被默认 `application.yml` 自动加载，避免开发环境误连现场设备。

## 3. 模块位置

公共网关：

- `com.railwaysim.localnet.LocalNetAdapter`
- `LocalNetAdapterManager`
- `LocalNetHealth`
- `RecentPacketStore`
- `ProtocolPacketAuditService`

信号：

- `com.railwaysim.localnet.signal.SignalDatabaseNodeFrameCodec`
- `SignalUdpAdapter`
- `SignalFrameDomainMapper`

司机台：

- 8080 `com.railwaysim.localnet.drivercab.DriverCabTcpAdapter`：管理 PLC、网络屏和信号屏现场 TCP；只接收 PLC 控制输入，屏幕链路仅发送显示帧。
- 8080 `DriverCabScreenPacketCodec` 和 26 字节 PLC 显示输出。
- 8080 `PeripheralFrameCodec`：保持 PLC 原始载荷并添加 RSIM Version 1 聚合头和 CRC32。
- 9300 `PeripheralAggregationController` + `DriverCabInputController` + `DriverCabPlcCodec` + `DriverCommandHolder` + `TrainStateHolder`：校验聚合方向、解码 PLC、缓存命令并写入车辆权威运行状态。

供电点表：

- `com.railwaysim.localnet.powerpoints.PowerPointDefinition`
- `PowerPointTableLoader`
- `LocalSimPowerPointDriver`
- `HttpSimPowerPointDriver`
- `PowerPointDomainMapper`

## 4. 联调入口

只读健康检查：

```http
GET /api/localnet/adapters
GET /api/localnet/adapters/{adapterId}
```

二进制/文本回放：

```http
POST /api/localnet/adapters/{adapterId}/replay
Content-Type: application/octet-stream
```

适配器 ID：

- `signal-udp`
- `driver-cab-tcp`
- `power-points`

`power-points` replay 使用文本 `pointId=value`，例如：

```text
ISO_P01_A_STATE=OPEN
```

## 5. 现场链路语义

信号 UDP：

- 支持完整 `0xff 0xf0/0xf1` 总控数据库节点帧。
- 支持实时 `0xff 0xf1` ADD、DELETE、CLEAR 生命周期包。
- 生命周期包进入 `TrainManager.applyLifecycleCommand()`。
- 信号周期状态首期按 shadow telemetry 记录，不覆盖中央联锁安全状态。

司机台 TCP：

- 公共标识固定为 `55 AA 55 AA`，WORD 使用小端。PLC 输入必须精确为 46 字节，头部总长/数据长必须为 `46/22`；PLC 输出为 `26/2`。任一标识字节或长度不符立即拒绝。
- PLC 绝对位偏移按现场表实现：byte 28 bit0 是紧急制动，byte 29 bit0..3 依次为开左/开右/关左/关右门，ATO 激活/自动折返激活在 byte 25 bit2/bit3。
- PLC 控制输入使用 46 字节 `DriverCabPlcCodec`。8080 TCP 适配器在 `EXTERNAL_HTTP` 模式下保持原始载荷、封装 RSIM 聚合帧后转发 9300，不写中央 `TrainManager`；9300 是命令仲裁和车辆状态变更的唯一执行端。
- PLC 显示输出继续使用 26 字节 `DriverCabPlcCodec`，由 8080 根据信号与中央状态编码。
- 网络屏和信号屏输出由 `DriverCabScreenPacketCodec` 从 `TrainState`、`SignalVehicleCommand.cabDisplay` 和供电状态生成。
- 网络屏、信号屏和视景系统均为纯输出显示端，不接收业务输入。中央只向8888/9999持续写TCP显示帧，并向8302发送UDP视景帧。
- 只有司机台PLC产生输入。8080保持原始46字节现场帧，增加 `RSIM` Version 1聚合头和CRC32后，通过9300单端口 `/vehicle-runtime/peripherals/frame` 转发；9300只接受 `PLC_INPUT` 通道。
- 9300 只接受已启动的列车实例；未知列车返回 404，非法枚举或级位返回 400。合法命令在下一 tick 与 MA、车门、自检和供电约束共同仲裁。

现场端点（补充文档修正后）：

| 设备 | 端点 | 平台角色 | 业务方向 |
|---|---|---|---|
| PLC | `192.168.100.123:8001` | TCP client | 接收 46 字节控制输入；发送 26 字节状态输出 |
| 网络屏 | `192.168.100.121:8888` | TCP client | 只发送 570 字节显示输出 |
| 信号屏 | `192.168.100.122:9999` | TCP client | 只发送 68 字节显示输出 |
| 视景系统 | `18.32.115.28:8302` | UDP sender | 只发送 Version 1.3 视景输出 |

站点 IP 不写入默认 `application.yml`；`config/localnet-site-example.yml` 已给出映射，现场部署显式启用。

供电点表：

- 点表文件支持 YAML 和 CSV；YAML 示例为 `config/power-point-table-example.yaml`。
- 点表字段包含 `pointId`、`name`、`direction`、`dataType`、`scale`、`address`、`domainTarget`、`defaultValue`、`quality`，并兼容 `targetType`、`targetId`、`targetField` 拆分写法。
- `READ` 点映射为合成 `PowerNetworkStateSnapshot`，用于联调与健康诊断。
- `WRITE` 或 `BIDIRECTIONAL` 点写入映射为 `PowerNetworkOperationRequest`，并通过 `PowerService.operate()` 进入现有供电操作链路。
- 当前实现提供 `LOCAL_SIM` 和 `HTTP_SIM` 两类驱动。未提供 Modbus TCP 或 OPC UA 驱动。

## 6. 验收建议

1. 默认配置启动后，`GET /api/localnet/adapters` 应显示适配器存在但不运行，且不绑定现场端口。
2. 使用 replay 验证信号生命周期包能增删列车。
3. 使用 9300 `POST /vehicle-runtime/peripherals/frame` 验证包含 `PLC_INPUT` 和原始 46 字节载荷的 RSIM 聚合帧被接受，并在下一 `step-fleet` 中体现到 `tractionCommand/brakeCommand/emergencyBrakeCommand`。
4. 回放现场帧 `55 AA 55 AA 2E 00 16 00 ...`，再分别破坏 4 字节标识、总长和数据长，确认合法帧接受且三类异常帧均拒绝。
5. 检查网络屏、信号屏和视景链路只有出站计数；向9300聚合入口提交任何显示输出通道应返回 `CHANNEL_DIRECTION_INVALID`。
6. 使用 8080 `GET .../state` 和 `GET .../plc-output` 验证信号侧状态投影与显示输出；结构化联调仍可向 8080 POST PLC 控制包，但它必须通过聚合入口转发且不得在中央执行。
7. 使用 `power-points` replay 验证点表写入能生成供电操作结果。
8. 机房启用真实链路后，检查 `inboundPackets`、`outboundPackets`、`errorPackets` 和最近报文摘要。
