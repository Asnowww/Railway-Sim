# 机房局域系统协议适配实施说明

## 1. 落地边界

现场协议按职责拆分到两个后端：中央/信号 `backend:8080` 保留信号 UDP、网络屏/信号屏和 PLC 显示输出；车辆管理 `vehicle-runtime-service:9300` 是 PLC 46 字节控制输入的唯一执行入口。中央不另起总控数据库节点服务。

实时报文收发不依赖数据库写入成功。`protocol_packet_log` 只做旁路诊断，写库失败时记录降级告警，现场链路继续运行。这不代表危险业务操作可绕过审计：故障注入/清除和供电操作必须先同步写 `operation_log`，失败返回 503。

协议源文件为 `docs/轨交多系统平台接口协议汇总20260630.docx`。

## 2. 启用方式

默认配置全部关闭：

```yaml
railway:
  simulation:
    localnet:
      enabled: false
      signal:
        enabled: false
      driver-cab:
        enabled: false
      power-points:
        enabled: false
```

机房联调时，把 `config/localnet-site-example.yml` 中需要的配置复制到本地 `application-local.yml`，再将对应链路改为 `enabled: true`。真实 IP、端口和点表不要写入 tracked `application.yml`。

公共网关：

- `com.railwaysim.localnet.LocalNetAdapter`
- `LocalNetAdapterManager`
- `LocalNetHealth`
- `RecentPacketStore`
- `ProtocolPacketAuditService`

信号：

- `com.railwaysim.localnet.signal.SignalDatabaseNodeFrameCodec`
- `SignalUdpAdapter`
- `SignalFrameDomainMapper`

司机台：

- 8080 `com.railwaysim.localnet.drivercab.DriverCabTcpAdapter`：信号侧状态投影、网络屏/信号屏与兼容回放，不是生产 PLC 控制命令源。
- 8080 `DriverCabScreenPacketCodec` 和 26 字节 PLC 显示输出。
- 9300 `DriverCabInputController` + `DriverCabPlcCodec` + `DriverCommandHolder`：解码、校验和缓存 46 字节控制命令。

供电点表：

- `com.railwaysim.localnet.powerpoints.PowerPointDefinition`
- `PowerPointTableLoader`
- `LocalSimPowerPointDriver`
- `HttpSimPowerPointDriver`
- `PowerPointDomainMapper`

## 4. 联调入口

只读健康检查：

```http
GET /api/localnet/adapters
GET /api/localnet/adapters/{adapterId}
```

二进制/文本回放：

```http
POST /api/localnet/adapters/{adapterId}/replay
Content-Type: application/octet-stream
```

适配器 ID：

- `signal-udp`
- `driver-cab-tcp`
- `power-points`

`power-points` replay 使用文本 `pointId=value`，例如：

```text
ISO_P01_A_STATE=OPEN
```

## 5. 现场链路语义

信号 UDP：

- 支持完整 `0xff 0xf0/0xf1` 总控数据库节点帧。
- 支持实时 `0xff 0xf1` ADD、DELETE、CLEAR 生命周期包。
- 生命周期包进入 `TrainManager.applyLifecycleCommand()`。
- 信号周期状态首期按 shadow telemetry 记录，不覆盖中央联锁安全状态。

司机台 TCP：

- PLC 控制输入使用 46 字节 `DriverCabPlcCodec`，生产数据必须送往 9300；8080 不再暴露 `POST .../plc-input`。
- PLC 显示输出继续使用 26 字节 `DriverCabPlcCodec`，由 8080 根据信号与中央状态编码。
- 网络屏和信号屏输出由 `DriverCabScreenPacketCodec` 从 `TrainState`、`SignalVehicleCommand.cabDisplay` 和供电状态生成。
- 9300 只接受已启动的列车实例；未知列车返回 404，非法枚举或级位返回 400。合法命令在下一 tick 与 MA、车门、自检和供电约束共同仲裁。

供电点表：

- 点表文件支持 YAML 和 CSV；YAML 示例为 `config/power-point-table-example.yaml`。
- 点表字段包含 `pointId`、`name`、`direction`、`dataType`、`scale`、`address`、`domainTarget`、`defaultValue`、`quality`，并兼容 `targetType`、`targetId`、`targetField` 拆分写法。
- `READ` 点映射为合成 `PowerNetworkStateSnapshot`，用于联调与健康诊断。
- `WRITE` 或 `BIDIRECTIONAL` 点写入映射为 `PowerNetworkOperationRequest`，并通过 `PowerService.operate()` 进入现有供电操作链路。
- 当前实现提供 `LOCAL_SIM` 和 `HTTP_SIM` 两类驱动。未提供 Modbus TCP 或 OPC UA 驱动。

## 6. 验收建议

1. 默认配置启动后，`GET /api/localnet/adapters` 应显示适配器存在但不运行，且不绑定现场端口。
2. 使用 replay 验证信号生命周期包能增删列车。
3. 使用 9300 `POST /api/vehicle/driver-cabs/{trainId}/plc-input` 验证 46 字节包被接受，并在下一 `step-fleet` 中体现到 `tractionCommand/brakeCommand/emergencyBrakeCommand`。
4. 使用 8080 `GET .../state` 和 `GET .../plc-output` 验证信号侧状态投影与显示输出，不得再向 8080 POST PLC 控制包。
5. 使用 `power-points` replay 验证点表写入能生成供电操作结果。
6. 机房启用真实链路后，检查 `inboundPackets`、`outboundPackets`、`errorPackets` 和最近报文摘要。
