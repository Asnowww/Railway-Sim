## 一、仿真控制（最简调试流程）

```
1. 确认后端在线 → http://localhost:8080/api/simulation/snapshot 返回 JSON
2. POST /api/simulation/reset  →  初始化仿真(TR-001/TR-002 两列车就位)
3. POST /api/simulation/tick   →  每调一次推进 200ms 仿真时间
4. 前端通过 WebSocket 自动收到更新后的 SimulationSnapshot
```

### 仿真控制 REST

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/simulation/snapshot` | 当前全量快照（不推进） |
| POST | `/api/simulation/tick` | 推进一个步长 → 返回全量快照 |
| POST | `/api/simulation/reset` | 重置仿真 |
| POST | `/api/simulation/start` | 启动仿真 |
| POST | `/api/simulation/pause` | 暂停仿真 |
| POST | `/api/simulation/fault/inject?segmentId=T02` | 注入区段故障 |
| POST | `/api/simulation/fault/clear?segmentId=T02` | 清除区段故障 |

## 三、SimulationSnapshot 结构与字段详解

### 3.1 顶层

```typescript
interface SimulationSnapshot {
  tick: number; simulatedTime: string;
  status: 'STOPPED' | 'RUNNING' | 'PAUSED';
  trains: TrainState[];
  trackSegments: TrackSegmentState[];      // 区段列表（含 fromNode / toNode / track）
  authorities: MovementAuthority[];        // 每车的 MA + currentSegmentId
  signalStates: SignalState[];            // 每区段入口信号灯色
  switchStates: SwitchState[];            // 道岔（只有有分叉的线路才非空）
  routeStates: RouteState[];              // 进路 AVAILABLE / ESTABLISHED / CONFLICTED
  powerSections: PowerSectionState[];
  alarms: Alarm[];
}
```

### 3.2 TrackSegmentState — 区段

```typescript
interface TrackSegmentState {
  id: string; startMeters: number; endMeters: number;
  speedLimitMetersPerSecond: number;
  occupancy: 'FREE' | 'RESERVED' | 'OCCUPIED' | 'FAULT';
  fromNode: string; toNode: string; track: string;
}
```

**颜色建议** — FREE `#3b4451` 灰 | RESERVED `#b8860b` 黄 | OCCUPIED `#dc2626` 红 | FAULT `#f97316` 橙(闪烁)

**拓扑规则** — 相同 `fromNode` + 不同 `track` = 分叉；相同 `toNode` = 汇合。道岔节点画紫色。

### 3.3 SignalState — 信号灯

```typescript
type SignalAspect = 'RED' | 'YELLOW' | 'GREEN';
interface SignalState {
  signalId: string; segmentId: string;
  positionMeters: number; aspect: SignalAspect;
  reasonTrainId: string | null;   // RED 时的原因
}
```

### 3.4 SwitchState — 道岔

```typescript
type SwitchPosition = 'NORMAL' | 'REVERSE';
interface SwitchState {
  id: string; nodeId: string;
  position: SwitchPosition; locked: boolean; activeSegmentId: string;
}
```

### 3.5 RouteState — 进路

```typescript
type RouteStatus = 'AVAILABLE' | 'ESTABLISHED' | 'CONFLICTED';
interface RouteState {
  routeId: string; status: RouteStatus;
  lockedSwitchIds: string[]; establishedByTrainId: string | null; axleSegmentIds: string[];
}
```

### 3.6 MovementAuthority — 移动授权

```typescript
interface MovementAuthority {
  trainId: string; authorityEndMeters: number;
  speedLimitMetersPerSecond: number; reason: string;
  currentSegmentId: string;   // 每车当前所在区段号
}
```

## 四、调度系统调试接口

### 4.1 时刻表

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/dispatch/plan` | 完整时刻表（PEAK/FLAT/OFF_PEAK 时段、发车间隔、停站时间） |
| GET | `/api/dispatch/plan/current` | 当前时段运行计划 |
| GET | `/api/dispatch/status` | 调度快照（运行模式、扰动、活跃命令） |
| GET | `/api/dispatch/disturbances` | 当前全部扰动事件（晚点、拥挤、间隔异常等） |
| GET | `/api/dispatch/commands` | 全部调度命令（含已执行/待执行/异常） |
| POST | `/api/dispatch/commands` | 手工下发调度命令 |

### 4.2 下发调度命令

```bash
curl -X POST http://localhost:8080/api/dispatch/commands \
  -H "Content-Type: application/json" \
  -d '{"trainId":"TR-001","commandType":"SPEED_LIMIT","detail":"8.0"}'
```

支持的 commandType：`HOLD`、`SPEED_LIMIT`、`TEMP_SPEED_LIMIT`、`SPEED_FACTOR`、`LIMIT_FACTOR`、`SPEED_BIAS`

### 4.3 进路管理

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/dispatch/route/list` | 全线路进路列表（routeId、起止站、状态、区段列表） |
| POST | `/api/dispatch/route/establish?routeId=R01&trainId=TR-001` | 为指定列车建立进路 |

### 4.4 站点记录

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/dispatch/station-records` | 各站列车到发事件记录 |

## 六、车辆故障注入调试

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/trains` | 全部列车列表 |
| GET | `/api/trains/{trainId}` | 单列车详情 |
| POST | `/api/trains/{trainId}/faults` | 注入车辆故障 |
| POST | `/api/trains/{trainId}/faults/clear` | 清除车辆故障 |

```bash
# 注入制动失效 → 安全距离从 120m → 180m → 观察 MA 缩短
curl -X POST http://localhost:8080/api/trains/TR-001/faults \
  -H "Content-Type: application/json" \
  -d '{"faultType":"BRAKE_FAILURE","reason":"debug"}'
```

## 七、前端 JS 调试代码

```typescript
const B = 'http://localhost:8080'

// —— 基础推进 ——
await fetch(`${B}/api/simulation/reset`, { method: 'POST' })
for (let i = 0; i < 10; i++) {
  const s = await fetch(`${B}/api/simulation/tick`, { method: 'POST' }).then(r => r.json())
  console.log(`tick ${s.tick}`, s.trackSegments.map(seg => seg.id + ':' + seg.occupancy).join(' '))
}

// —— 故障注入 ——
await fetch(`${B}/api/simulation/fault/inject?segmentId=T02`, { method: 'POST' })
// T02 变橙色 FAULT → 前方信号 SIG-T02 变 RED → TR-001 的 MA 截断到故障点前

// —— 查进路 ——
const routes = await fetch(`${B}/api/dispatch/route/list`).then(r => r.json())
console.table(routes, ['routeId','status','fromStation','toStation'])

// —— 手工建进路 ——
await fetch(`${B}/api/dispatch/route/establish?routeId=R01&trainId=TR-001`, { method: 'POST' })

// —— 发调度命令 ——
await fetch(`${B}/api/dispatch/commands`, {
  method: 'POST', headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ trainId: 'TR-001', commandType: 'SPEED_LIMIT', detail: '8.0' })
})

// —— 车辆故障 → 安全距离 ——
await fetch(`${B}/api/trains/TR-001/faults`, {
  method: 'POST', headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ faultType: 'BRAKE_FAILURE', reason: 'debug' })
})
```

## 九、常见调试场景

| 场景 | 操作 |
|------|------|
| 看 MA 是否正常 | tick 后读 `authorities` → 检查 `authorityEndMeters` 和 `reason` |
| 看区段颜色 | tick 后读 `trackSegments[].occupancy` |
| 道岔/进路 | 切到 `line-branch-demo.yaml` → tick → 看 `switchStates` / `routeStates` |
| 故障→信号联锁 | 注入故障 → tick ×10 → 看信号变 RED + MA 截断 |
| 调度命令生效 | 发 `SPEED_LIMIT` → tick → 看 `authorities[].speedLimitMetersPerSecond` 变化 |
