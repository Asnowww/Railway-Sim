## 一、数据获取方式

### REST API（主动查询/控制）

| 方法 | URL | 说明 |
|------|-----|------|
| GET | `/api/simulation/snapshot` | 获取当前全量快照 |
| POST | `/api/simulation/tick` | 推进一个仿真步长（返回快照） |
| POST | `/api/simulation/reset` | 重置仿真 |
| POST | `/api/simulation/fault/inject?segmentId=T02` | 注入区段故障 |
| POST | `/api/simulation/fault/clear?segmentId=T02` | 清除区段故障 |

### WebSocket（被动推送）

```
ws://localhost:8080/ws/simulation
消息格式: { "type": "snapshot", "payload": SimulationSnapshot }
```

## 二、SimulationSnapshot 顶层结构

```typescript
interface SimulationSnapshot {
  tick: number                    // 仿真步数
  simulatedTime: string           // ISO时间戳
  status: 'STOPPED' | 'RUNNING' | 'PAUSED'
  trains: TrainState[]            // 列车状态列表
  trackSegments: TrackSegmentState[]  // 区段状态列表
  authorities: MovementAuthority[]    // 移动授权列表
  signalStates: SignalState[]         // 信号灯状态列表
  switchStates: SwitchState[]         // 道岔状态列表 ← 新增
  routeStates: RouteState[]           // 进路状态列表 ← 新增
  powerSections: PowerSectionState[]  // 供电分区(方首岳)
  alarms: Alarm[]                     // 告警列表
}
```

## 三、信号轨道相关字段详解

### 3.1 TrackSegmentState — 区段（轨道条）

```typescript
interface TrackSegmentState {
  id: string                     // 区段ID，如 "T01", "T02"
  startMeters: number            // 起点公里标(m)
  endMeters: number              // 终点公里标(m)
  speedLimitMetersPerSecond: number  // 线路限速(m/s)
  occupancy: 'FREE' | 'RESERVED' | 'OCCUPIED' | 'FAULT'  // 占用状态
  fromNode: string               // 起点节点ID，如 "N1" ← 拓扑用
  toNode: string                 // 终点节点ID，如 "N2" ← 拓扑用
  track: string                  // 轨别，如 "main"/"branch"/"depot"
}
```

**occupancy 前端颜色建议：**

| 值 | 颜色 | 含义 |
|----|------|------|
| FREE | `#3b4451` 灰 | 空闲 |
| RESERVED | `#b8860b` 黄 | 已被MA预留，其他车不能占 |
| OCCUPIED | `#dc2626` 红 | 有车在上 |
| FAULT | `#f97316` 橙(闪烁) | 故障 |

**拓扑渲染方式：**
- `fromNode` → `toNode` 构成有向边
- 相同 `fromNode` 且 `track` 不同 = 分叉
- 相同 `toNode` = 汇合
- 示例：T02.fromNode==T03.fromNode=="N2" → N2是道岔分叉点

### 3.2 SignalState — 信号灯

```typescript
type SignalAspect = 'RED' | 'YELLOW' | 'GREEN'

interface SignalState {
  signalId: string               // 信号机ID，如 "SIG-T01"
  segmentId: string              // 所在区段ID
  positionMeters: number         // 公里标
  aspect: SignalAspect           // 灯色
  reasonTrainId: string | null   // RED时的原因（列车ID或"FAULT"）
}
```

灯色规则：区段OCCUPIED/FAULT→RED | RESERVED→YELLOW | FREE→GREEN

### 3.3 SwitchState — 道岔（新增）

```typescript
type SwitchPosition = 'NORMAL' | 'REVERSE'

interface SwitchState {
  id: string                     // 道岔ID，如 "SW01"
  nodeId: string                 // 所在节点ID，如 "N2"
  position: SwitchPosition       // 当前开通方向
  locked: boolean                // 是否锁死（信号开放后=锁死，禁止扳动）
  activeSegmentId: string        // 当前激活的区段ID
}
```

### 3.4 RouteState — 进路（新增）

```typescript
type RouteStatus = 'AVAILABLE' | 'ESTABLISHED' | 'CONFLICTED'

interface RouteState {
  routeId: string                // 进路ID
  status: RouteStatus            // 状态
  lockedSwitchIds: string[]      // 该进路锁死的道岔ID列表
  establishedByTrainId: string | null  // 占用该进路的列车(null=空闲)
  axleSegmentIds: string[]       // 核心区段列表（冲突检测用）
}
```

### 3.5 MovementAuthority — 移动授权

```typescript
interface MovementAuthority {
  trainId: string
  authorityEndMeters: number     // MA终点公里标（车最远能开到这）
  speedLimitMetersPerSecond: number  // 允许最大速度
  reason: string                 // 限制原因：前车限速/故障降级/进路冲突/前方区段空闲
}
```

## 四、可用线路配置

修改 `application.yml` 中 `line-data-path` 切换线路：

| 文件 | 规模 | 拓扑类型 |
|------|------|---------|
| `line-simple-demo.yaml` | 4段/5站/0道岔 | 简单直线 |
| `line-branch-demo.yaml` | 7段/1站/2道岔 | 正线+侧线分合流 |
| `line-loop-demo.yaml` | 9段/4站/1道岔 | 环线+车辆段出入线 |
| `line-yard-demo.yaml` | 16段/4站/5道岔 | 终点站咽喉区+3站台 |
| `line-ggz.yaml` | 35段/1站 | GGZ站真实工程数据 |

每段都带 `fromNode`/`toNode` 拓扑信息，可直接渲染成网络图。

## 五、你的前端需要展示/交互的内容

### 轨道区段视图
- 根据 `trackSegments` 绘制区段条，颜色按 `occupancy`
- 用 `fromNode→toNode` 连接关系绘制拓扑连线
- 道岔节点画分叉/汇合（参考 `switchStates`）

### 信号灯视图
- 每个区段入口一个灯泡，颜色按 `aspect`
- RED时显示 `reasonTrainId` 作为悬浮提示

### 列车位置
- 从 `trains[].positionMeters` 算像素位置
- 车头=positionMeters，车尾=positionMeters-lengthMeters

### MA范围
- 从列车位置到 `authorities[].authorityEndMeters` 画半透明条

### 交互按钮
- Tick单步 / 自动推进 / Reset / 注入故障
