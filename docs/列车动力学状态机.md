# 列车动力学状态机

本文档描述车辆动力学层在 `database/线路数据(1).xls` 约束下的状态机。该层位于 `OnboardTrainSubsystem`，由 `OnboardTrainSubsystemManager` 负责调用本地或外部车辆控制节点，把线路、信号和供电约束转换为车辆物理模型的牵引/制动命令；调度约束先由信号模块折算为 MA/限速。

## XLS 约束摘要

当前默认线路源为数据库。启动时 `DatabaseLineDataImporter` 会把 `../database/线路数据(1).xls`
导入静态线路表，随后 `JdbcLineDataLoader` 从数据库重建 `OperationalLineData`。车辆层可消费以下约束：

| 约束 | 当前数据规模 | 车辆层用途 |
|---|---:|---|
| 轨道区段 | 319 段，线路长度约 47.5 km | 定位列车所在区段和剩余线路空间。 |
| 静态限速 | 238 条，约 7.77-27.77 m/s | 决定牵引、巡航、超速制动状态。 |
| 坡度 | 182 条 | 进入制动距离估算和物理模型阻力项。 |
| 车站/站台 | 13 站、56 站台 | 决定进站制动和站停状态。 |
| 信号/应答器/进路 | 157 信号、374 应答器、249 进路 | 当前主要通过信号授权 MA 间接影响车辆。 |
| 道岔 | 60 组 | 当前通过分段限速和线路拓扑间接影响车辆。 |

## 状态优先级

状态机按安全优先原则从高到低决策：

1. `SELF_CHECK_BLOCKED`：车门未锁闭、自检失败或制动不可用，切除牵引并施加制动。
2. `SAFETY_BRAKE`：移动授权耗尽，进入 ATP 安全制动。
3. `POWER_LOSS`：受流不可用或供电功率为 0，切除牵引并制动。
4. `MA_BRAKE`：移动授权距离不足以覆盖服务制动距离和安全缓冲，按距离缺口制动。
5. `STATION_STOPPED`：列车已低速进入站停窗口，保持停稳制动。
6. `STATION_BRAKE`：基于 XLS 车站距离进入进站制动。
7. `OVERSPEED_BRAKE`：当前速度超过线路/信号综合限速。
8. `POWER_DERATED`：供电可用但牵引能力折减。
9. `OVERLOAD_DERATED`：动态载重超限或牵引单元不足，牵引命令降额。
10. `ACCELERATING`：低于目标速度且牵引空间充足。
11. `CRUISING`：接近目标速度，低牵引巡航。
12. `COASTING`：达到目标速度，惰行。

## 动力学判据

车辆层每个 tick 计算：

```text
speedLimit = min(signalSpeedLimit, trackSpeedLimit)
maDistance = movementAuthorityEnd - trainPosition
stationDistance = nextStationCenter - trainPosition
stoppingDistance = v^2 / (2 * effectiveServiceDecel(loadMass, availableBrakeCount))
```

坡度来自 XLS。为避免异常大坡度直接放大控制制动距离，状态机只在制动距离估算中把坡度贡献限制在 `[-0.04, 0.04]`；车辆物理模型仍消费原始坡度，保留数据约束对加速度的影响。

## 状态输出

状态机输出已贯通到 `TrainState`、WebSocket 类型和 MySQL 快照：

| 字段 | 含义 |
|---|---|
| `dynamicsState` | 当前动力学状态。 |
| `dynamicsConstraintReason` | 触发该状态的主要约束。 |
| `speedLimitMetersPerSecond` | 综合限速。 |
| `movementAuthorityDistanceMeters` | MA 剩余距离。 |
| `stationDistanceMeters` | 下一站中心距离。 |
| `stoppingDistanceMeters` | 当前速度下的服务制动距离估计。 |

数据库侧新增列记录在 `database/2026-07-07-train-dynamics-state-machine.sql`。xls 落库和数据库源切换记录在 `database/2026-07-07-line-data-database-source.sql`。规范表定义维护在 `database/schema.sql`。
