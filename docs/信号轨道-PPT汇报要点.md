# 信号轨道模块 — PPT汇报要点

## 核心贡献

### 1. M9 真实线路接入 & 区段优化
- 北京地铁9号线 13站/24上段+23下段，来自真实信号机公里标数据
- 长段（4.6km/5.5km）按站间距切短，前端信号粒度可达 150m~1.9km

### 2. 上下行双线独立运行
- UP轨 + DOWN轨 并行，里程重叠但轨道隔离
- 同轨前向邻居图（kmForwardMap）：不依赖 from→to 节点方向，上下行通用
- 跨轨占用/预留/MA 互不干扰

### 3. 调度→信号→车辆 发车闭环
- 移除硬编码初始车，全部走调度 DEPART 命令
- DOWN 门禁删除，下行方向独立发车
- 列车创建后显式绑定轨道，避免并行股道里程重叠误判

### 4. 三层命令反馈闭环
- 信号命令接收回执（POST /command-receipt）
- MA/限速应用反馈（每 tick 自动生成 feedback）
- 效果确认（约束来源：SIGNAL_MA / FRONT_TRAIN / STATION_DWELL）

### 5. 占用/预留/联锁 联动
- overlaps 区间交集修复（走过释放）
- MAIN 进路多车追踪 hold 自动清除
- 拓扑障碍异轨即停搜
- 预留段严格同轨过滤（不溢到对向）

## 可提供的素材

| 类型 | 内容 |
|------|------|
| **截图** | 上下行双线态势图（红/黄色占用带）、调度命令面板（带反馈状态）、信号轨道调试视图（进路生命周期事件） |
| **视频/录屏** | 四车按时刻表发车、车走轨道染色跟随、站停→MA释放→区间预留流转 |
| **架构图** | 调度→信号→联锁→MA→车辆 链路、反馈三层模型、YAML数据流程 |

## 一句话总结
> **信号轨道模块实现了北京M9真实线路上下行双线运行、调度全自动发车闭环、以及从命令下达到效果确认的三层反馈，占用/预留/联锁联动，前端红黄色跟随列车动态流转。**
