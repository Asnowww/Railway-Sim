## 三、关键成果与产出

1. **M9 真实线路全线切分**：原始 25 段信号闭塞区间（最长 5.5km）按 13 站中心公里标切分为 47 段，上行 24 段 + 下行 23 段，典型段长 0.8~1.9km。新增 22 个站间节点，进路 `axle_section_ids` 和信号 `segment_id` 自动映射
2. **kmForwardMap 同轨前向邻居图**：基于公里标数值邻接（`next.start ≈ this.end` + 同轨过滤），不依赖 YAML 的 from→to 节点方向。`resolveTopologyObstacle` 和 `collectReservedAlongActiveTopology` 全部切换，跨轨互扰问题根治
3. **四层轨道过滤体系**：`segmentAt()` main>up>down → `segmentForTrain()` 历史轨道优先 → `resolveTopologyObstacle()` 异轨即停 → `collectReservedAlongActiveTopology()` 渡线即停，上下行在重叠 km 空间完全隔离
4. **调度全自动发车**：删除 `TrainManager` 中的 `addInitialTrain` 硬编码、删除 `handleDepartures` 的 `toPos < fromPos` DOWN 门禁、`assignTrainToTrack` 改为无条件绑定。16 车 50s 间隔全上行走廊按调度计划自动发车
5. **三层反馈闭环**：`POST /api/signal-track/command-receipt`（接收回执）+ `generateTimeCommandFeedback()` 每 tick 自动推 feedback + `checkDispatchCompletion()` 效果确认 → dispatchService.acceptFeedback
6. **六项关键修复**：overlaps 区间交集（`tail<end && head>start`）、进路 hold 自动清除、终点释放、进站制动保护、异轨 MA 不互相阻挡、预留同轨严格过滤
7. **文档交付**：详细设计文档 16 章（含架构图、算法描述、状态机、API 参考）、问题排查补充文档 4 节、周报 2 份。34 个单元测试全部通过

## 四、遇到的问题与解决方案

| 序号 | 问题描述 | 解决方案 / 当前进展 | 状态 |
|------|---------|------------------|------|
| 1 | 上下行 km 重叠导致列车 MA 跨轨截断：列车从 up 轨沿 forwardNeighbor 搜索到 down 轨上的车，MA 被截成零 | 加 `kmForwardMap` 同轨邻接 + `resolveTopologyObstacle` 异轨即停。搜索到 crossover/异轨段立即返回 INFINITY，不再沿渡线跳到对向 | 已解决 |
| 2 | 列车驶过后区段不释放，全线飘红 | `overlaps()` 原判断为严格包含 `head > start && tail < end`，车尾刚离段仍判为占。修正为正确区间交集 `tail < end && head > start` | 已解决 |
| 3 | 下行车被门禁拦截：SVC-003(DOWN) 的 `toPos < fromPos` 导致发车跳过 | 删除该门禁逻辑，改为无条件发车 + 显式轨道绑定 | 已解决 |
| 4 | 下行车跑到上行轨道：km 0 处 U02 和 D03 同时存在，段回退选了 up | `handleDepartures` 中将 `assignTrainToTrack` 改为无条件执行（UP→"up"，DOWN→"down"） | 已解决 |
| 5 | 多车在同一进路口被锁死："等待进路建立(ROUTE_NOT_LOCKED)" 永不释放 | `touchRoutes` 中新增 MAIN 进路 OCCUPIED 时自动清除 hold 的逻辑 | 已解决 |
| 6 | 物理引擎无法递减公里标，下行方向不能直接跑反向 | 对比了 from↔to 交换、LD 段追加、环路 km 偏移三种方案，均因下游路由验证或拓扑链断裂无法直接使用。当前采用快照层镜像方案，内部全用物理递增 km | 跟进中 |
