# 北京地铁供电架构抽象与仿真建模

## 1. 资料依据

公开资料可抽象出以下建模事实：

- 北京地铁官方“设备知识-供电”说明北京地铁供电系统包含 `10kV` 电源系统、`750V` 直流牵引供电系统、`400V` 低压配电系统和应急照明系统；牵引供电采用接触轨，运行方式包括双边或单边供电。
- 城轨供电系统资料普遍将系统分成主变电所/城市电网、中压环网、牵引变电所、直流接触网/接触轨、钢轨回流、杂散电流防护和电力监控系统。
- 北京换乘站供电能力研究中，以车公庄站为例搭建中压环网仿真，使用开闭所、`10kV` 两段母线、联络开关、进线电流、母线压降和跨线支援供电作为核心分析对象。

## 2. 软件抽象

本项目的外部供电仿真系统按四层建模：

| 层级 | 软件对象 | 建模职责 |
|---|---|---|
| 城市电网与中压环网 | `MediumVoltageBus`、`RingFeeder` | 表示 `10kV` 母线、环网馈线、进线电流、母线压降、馈线过载。 |
| 牵引变电所 | `SubstationState`、`DeviceState` | 表示牵引变压器、整流器、直流快速断路器和设备可用性。 |
| 直流牵引网 | `ThirdRailState`、`IsolatorState` | 表示 `750V` 接触轨分区、隔离开关、双边/单边/越区供电和接触轨电压。 |
| 回流与杂散电流 | `StrayMonitorState` | 表示排流柜状态、极化电位、回流异常和腐蚀风险等级。 |

## 3. 仿真规则

- 正常工况下，接触轨分区按 `DOUBLE_END` 双边供电处理。
- 当本分区牵引变电所不可用，但相邻区段仍可支援时，模型切换为 `SINGLE_END` 或 `CROSS_FEED`。
- 隔离开关断开时，对应接触轨区段进入隔离/失电状态。
- 直流快速断路器 `TRIPPED` 或 `OPEN` 时，对应分区退出正常供电。
- 接触轨电压按区段净电流、接触轨阻抗、钢轨回流阻抗和供电方式折减计算。
- 中压侧按牵引负荷折算进线电流，并给出母线压降比例；馈线电流超过热稳定限值时生成告警。
- 杂散电流风险由排流柜状态、区段隔离状态和牵引回流电流共同决定，输出 `NORMAL`、`ATTENTION`、`WARNING`、`CRITICAL`。

## 4. 服务接口

外部仿真服务位于 `power-network-service/`，保留中央系统需要的接口：

- `POST /power-network/bootstrap`
- `GET /power-network/state`
- `GET /power-network/events`
- `POST /power-network/operations`

同时新增建模验证接口：

- `GET /power-network/topology`
- `POST /power-network/state/query`

`state/query` 用于注入区段牵引负荷、再生功率和取流电流，使外部服务能够独立完成地铁电网状态推演。
