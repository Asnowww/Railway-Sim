## 1. 升级目标与代际边界

现有`TrainTractionBrake/1.0.0`是标量模型：使用最大牵引力、3.2 MW机械功率上限、固定Davis系数和固定再生比例。目标版本升级为`TrainTractionBrake/2.0.0`，使用52点电机轴侧牵引/制动转矩曲线、6.5传动比、0.46 m车轮半径、六车编组质量和动态Davis公式。

本次升级完成后，系统必须满足：

1. 同一曲线参数集由9300 Java fallback和9000真实FMU共同加载，两个执行器在曲线节点及节点中间保持数值一致。
2. 车辆长度、空车质量和载荷工况不再使用120 m、198 t和72 t旧基线。
3. 牵引力由电机转矩曲线换算，供电能力和黏着条件只作为附加上限。
4. 制动曲线表示最大电制动能力；常用制动采用电制动优先、空气制动补足，紧急制动使用纯空气制动。
5. 正常`POST /step-fleet`保持向后兼容；静态曲线只在服务启动和FMU实例初始化阶段装载。
6. 9200继续只消费实际受流功率、回馈功率和网流，不读取电机曲线。

### 2.2 效率建模约束

`0.882`已经按`4336 kW / (1500 V × 3277.5 A)`定义为系统总效率。首版不得再把齿轮箱95%～98%的经验效率单独乘入功率或牵引力，否则发生重复计损：

```text
motorSpeedRpm = vehicleSpeedMps / wheelRadiusMeters
                * gearRatio * 60 / (2*pi)

curveForceNewtons = interpolatedMotorTorqueNm
                    * motorCount * gearRatio / wheelRadiusMeters

mechanicalTractionPowerWatts = tractionForceNewtons * vehicleSpeedMps
tractionPowerWatts = mechanicalTractionPowerWatts / tractionTotalEfficiency
```

齿轮箱分项效率只作为后续可选标定项。只有同时取得逆变器、电机和齿轮箱分项效率，并重新校核曲线功率口径后，才允许拆分`tractionTotalEfficiency`。

### 2.3 制动策略冻结

- 52点制动转矩曲线表示最大电制动能力，不表示总制动力。
- 常用制动目标总力：`F_service_demand = brakeCommand × trainMassKg × 1.0`。
- 曲线电制动力、总效率和9200再生吸收预算共同决定实际再生力。
- `F_air = max(0, F_service_demand - F_regen)`，保证低速或电网拒绝再生时制动需求不消失。
- 紧急制动目标总力：`F_emergency = trainMassKg × 1.3`；牵引归零、再生归零、全部由空气制动承担。
- 速度为零时制动力不得导致速度反向；停车保持继续由9300状态机负责。

### 3.3 校验规则

1. 七个曲线数组长度必须全部为52。
2. `speedRpm`单调不减；只允许资料中第52点与第51点构成唯一重复终点，且所有关联曲线在这两个索引上的值必须相同。其余数组有限且非负。
3. `motorCount=16`、`axleCount=24`、`gearRatio>0`、`wheelRadiusMeters>0`。
4. AW0、AW2、AW3整列质量必须等于对应单车质量按编组求和。
5. `AW0 < AW2 < AW3 <= hardMassLimitKg`，每种单车质量不得超过56 t。
6. 对每个有效节点校验`torque × rpm × 2π / 60 / 1000`与参考机械功率，误差不超过0.5%。
7. 在1500 V基准下校验峰值牵引效率接近0.882、峰值制动效率接近0.802；误差超过1%时拒绝启动。
8. `parameterSetId`继续使用YAML原始字节SHA-256；9300、9000必须一致。

## 4. UP2：Modelica曲线动力学与Java等价回退

### 4.1 牵引计算顺序

```text
rpm = v / r * i * 60 / (2*pi)
T_curve = linearInterpolate(speedRpm, tractionTorqueNmPerMotor, rpm)
F_curve = T_curve * motorCount * i / r
F_command = clamp(tractionCommand, 0, 1) * F_curve
F_adhesion = adhesionCoefficient * trainMassKg * g
P_supply_mech = powerAvailableWatts * tractionTotalEfficiency
F_supply = P_supply_mech / max(v, 0.5)
F_traction = min(F_command, F_adhesion, F_supply)
P_mech = F_traction * v
P_grid = P_mech / tractionTotalEfficiency
I_grid = P_grid / max(lineVoltageVolts, 1)
```

低于切除电压、受流不可用或`powerAvailableWatts<=0`时，牵引输出仍归零。不得再保留3.2 MW硬上限；曲线与供电能力共同形成新的功率上限。

### 4.2 动态Davis阻力

FMU按资料原式计算，`M=trainMassKg/1000`、`vKph=speedMetersPerSecond×3.6`：

```text
F_resistance = 6.4*M + 130*24 + 0.14*M*vKph
             + (0.046 + 0.0065*(6-1))*10.6*vKph^2
```

不得把空车等效的4560、113.4、10.784016固定为所有载荷工况的Davis系数。

### 4.3 电制动与空气制动混合

```text
T_brake = linearInterpolate(speedRpm, brakeTorqueNmPerMotor, rpm)
F_electric_curve = T_brake * motorCount * gearRatio / wheelRadius
F_total_demand = emergency
  ? trainMassKg * 1.3
  : clamp(brakeCommand, 0, 1) * trainMassKg * 1.0

F_regen_candidate = emergency ? 0 : min(F_total_demand, F_electric_curve)
P_regen_candidate_mech = F_regen_candidate * v
P_regen_grid_limit_mech = regenPowerAvailableWatts / regenTotalEfficiency
P_regen_mech = min(P_regen_candidate_mech, P_regen_grid_limit_mech)
F_regen = min(F_regen_candidate, P_regen_mech / max(v, 0.5))
P_regen_grid = F_regen * v * regenTotalEfficiency
F_air = F_total_demand - F_regen
```

合力只扣除一次`F_total_demand`。新增诊断量可以报告再生力和空气制动力，但不得改变现有`brakeForceNewtons`表示总制动力的语义。

### 4.4 实施与验收

- Modelica曲线使用FMI固定参数数组；插值区间外采用端点钳制，不允许线性外推成负转矩。达到最高转速时直接返回末值，不进入第51～52点的零宽插值区间。
- Java fallback使用相同数组、相同插值函数、相同限幅顺序和相同低速保护。
- [验收条件] 52个节点和51个中点上，FMU与Java的牵引力、电制动力及功率相对误差不超过0.5%。
- [验收条件] AW0/AW2/AW3下阻力、加速度和制动减速度均通过解析断言。
- [验收条件] 零速、83.2 rpm附近、基速点、最高转速、欠压、供电限功率、黏着受限和再生预算为0均无跳变异常或除零。
- 回滚：通过上一代9300/9000镜像标签和对应参数挂载整体切回`SCALAR_V1`；当前v2镜像内的`JAVA_FALLBACK`与FMU均使用同一Curve v2参数，禁止跨代混装。

## 7. UP5：9200容量复核与车电闭环回归

9200接口不新增电机曲线字段，但按新曲线重新核验：

1. 单车峰值受流功率参考约4.916 MW，不能继续按3.2 MW电网侧上限设计测试。
2. 单车峰值参考回馈功率约5.464 MW，实际值仍受同供电段吸收预算约束。
3. 供电段容量不足时必须通过`powerAvailableWatts`限制下一周期车辆牵引力，FMU不得再重复按网压降额。
4. 同段列车A增加负载后，列车B下一周期的机械功率、牵引力或加速度至少一项下降；不同段无联供时不受影响。
5. 再生预算为0或低速电制动不足时，总制动力由空气制动补齐。

[验收条件] 9200负荷守恒、分区耦合、未吸收回馈和一周期延迟测试全部通过；现有供电拓扑和API保持兼容。

- v1与v2分别保留完整的车辆参数、FMU、映射和镜像标签，回滚必须整代执行。
- 任一阻断项失败时，将9300固定到已验收的v1 Java fallback或v1 FMU，不允许v2参数加载到v1执行器。
- 不回滚数据库，不新建MySQL容器，不修改`App.vue`。
